Process & Decision Log

Key Design Choices

  1. Deterministic scoring first: Chosen over free-form LLM scoring to reduce recruiter trust issues and simplify QA.
  2. n8n for orchestration: Selected for low-code automation, clear visual workflow, and rapid SMB deployment.
  3. Single endpoint contract: /api/screen kept minimal to speed frontend/backend integration.
  4. Explainability by default: Every result includes summary, strengths, and gaps to support final human judgment.

Prompt Iteration Notes

  • Iteration 1: Open-ended evaluation prompt produced inconsistent scoring.
  • Iteration 2: Added strict weights and integer scoring; consistency improved significantly.
  • Iteration 3: Enforced strict JSON schema + explicit action labels (Interview/Hold/Reject).
  • Iteration 4: Added missing-skills and strengths extraction to improve recruiter readability.

What We Learned

  • SMB users prioritize speed and clarity over model sophistication.
  • Transparent scoring creates stronger adoption than black-box ranking.
  • Most operational delays happen before interviews, so triage automation has immediate ROI.

Tradeoffs Accepted

  • Resume parsing is text-focused in MVP; advanced PDF/OCR handling deferred.
  • Bias and fairness analytics are flagged for next phase, not fully solved in week 1.
  • ATS integrations postponed to keep MVP deployable and demo-ready quickly.
